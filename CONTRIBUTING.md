# Contributing to Communication Growth Skill

感谢你的兴趣！这个 Skill 的核心是**帮助人看清自己的沟通真相**。如果你认同这个理念，欢迎贡献。

## 贡献类型

### 1. 新场景示例
- 添加新的对话场景（如：跨文化沟通、亲子对话、客户谈判、面试场景）
- 每个示例需要包含：原始对话 + 分析报告 + 修正方案
- 放在 `examples/` 目录，命名格式：`example-{scene}-{subscene}.txt/md`

### 2. 缺陷分类扩展
- 发现现有 10 大类别覆盖不到的沟通缺陷？
- 提供：缺陷名称、定义、典型表现、至少 2 个真实案例
- 在 P3 中补充分类，在 P4 中补充修正方法论

### 3. 方法论补充
- 发现新的沟通理论/技巧/框架？
- 提供：理论名称、来源、适用场景、具体话术模板
- 在 P4 的方法论库中补充

### 4. 工具改进
- Python 工具的 bug 修复、性能优化、新功能
- 保持代码简洁，注释清晰，符合 PEP 8

### 5. 跨文化适配
- 当前版本以中文语境为主
- 欢迎添加英文版 prompt、其他文化语境的变体

## 提交规范

1. **Fork 仓库** → 创建 feature 分支 → 提交 PR
2. **PR 标题格式**：`[类型] 简要描述`，如 `[示例] 添加客户谈判场景`
3. **PR 描述**必须包含：
   - 改动动机
   - 具体改动内容
   - 测试方式（如果有代码）
4. **保持风格一致**：
   - 拒绝老好人、拥抱冲突、客观理性、方法论导向
   - 不鸡汤、不安慰、不贴标签

## 代码风格

- Python: PEP 8, type hints 优先
- Markdown: 80 字符行宽（代码块除外）
- Prompt: 中文为主，术语保留英文

## 讨论区

- 有想法但不确定怎么实现？开 Issue 讨论
- 发现了某个沟通缺陷但不知道怎么分类？开 Issue 讨论
- 想分享自己的沟通复盘？欢迎！

## 行为准则

- 对事不对人
- 沟通缺陷 ≠ 人格缺陷
- 尖锐的反馈 > 温和的废话
